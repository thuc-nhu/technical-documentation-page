Introduction

What is Python?

Python was created by Guido van Rossum and released in 1991. The primary goal of Python is readability and simplicity. It allows programmers to write more readable code with fewer lines.

This documentation will go over the five major data types categories inherent in Python.

Intended Audience

This documentation is for programmers who are beginning to use Python. A basic understanding of how to write code using Python is required.

Variables

A variable is a symbolic name given to some known or unknown data. Variables stores data so that a program can easily access that data or use it later. Variables are often described as labels or envelopes that store information.

Naming Variables

Variables can be named using the following rules.

Variable names:

  • Can contain upper or lowercase letters, numbers, and underscores.
  • Must begin with a letter or an underscore.
  • Can't start with a number.
  • Can't contain spaces.
  • Shouldn't contain Python keywords and function names.
  • Should be short and descriptive.
  • Are case sensitive.

The following are examples of acceptable variable names.

message_1 greeting_message n first_name

The following are not acceptable variable names.

1_message greeting message length_of_persons_name

Note length_of_persons_name is a valid variable name but long and cumbersome. A better variable is name_length.

Data Types

In programing, data types help the computer understand and interact with the data. In Python, there are five built-in data type categories.

  • Numbers: int, float, complex
  • Sequences: string, list, tuple
  • Mapping: dict
  • Set
  • Boolean: bool

Immutable vs mutable data types

Data types can either be immutable or mutable. Mutable data types can change their content once created. However, immutable data types' content can't change.

The following are immutable data types:

  • int
  • float
  • bool
  • str
  • tuple

The following are mutable data types:

  • list
  • set
  • dict

It is easier to think of mutability by thinking of a list. Consider your grocery list. Let's say that you need lettuce, bread, and eggs this week. You add them to your grocery list. Here is what your list looks like:

  • lettuce
  • bread
  • eggs

You relook at what you have at home and find that you don't need lettuce but need milk and apples. You remove lettuce from your list and add milk and apples. Here is your grocery list now:

  • bread
  • eggs
  • milk
  • apples

You only have one grocery list. The items on the list can change, be removed, or new items can be added.

For the number 56, if you change any digit of the number, it will be a different number altogether.

Get data types from a pre-existing variable

Python automatically assigns variable data types. Once you write x = 5, then x has the data type of int. Get an object's data type by using the method type(). Change an object's data type by recasting it.

Get the data type of any object by using the type() function. For example, type(5) will return int and type(5.0) will return float.

The code below is an example for getting the data type of a variable.

x = 5 type(x)

Set variable data type

Python automatically assigns varaible data types. There may be a time that you want to stipulate a data type to a variable. Specify a variable's data type by using the int(), float(), or str() constructor when assigning values to variables. For example:

x = str("Hello") y = str(5)

The above code will assign the data type string to the variables x and y even if 5 is an integer.

Changing variable data types (casting)

You may want to change a variable's data type. Specifying the type of data by using constructor functions will recast the variable into the desired data type.

  • int() - Casts any integer, float, or string into an integer data type.
  • float() - Casts any integer, float, or string into a float data type.
  • str() - Casts any data into a string data type.

Using int()

The int() constructor casts any integer, float, or string into an integer data type. For example:

x = int(1) y = int(2.8) z = int("3")

The above code will give the following values:

x = 1 y = 2 z = 3

Note that the value for y is 2. The int() constructor does not round a number; it ignores the decimal part.

The int() constructor will only cast strings into integer data types if the string represents a whole number. The following code will return an error.

x = int('three') y = int("3.8")

Using float()

The float() constructor casts any integer, float, or string into a float data type. For example:

x = float(1) y = float(2.8) z = float("3") w = float("4.2")

The above code will give the following values:

x = 1.0 y = 2.8 z = 3.0 2 = 4.2

Note that the float() constructor will only cast strings into float data types if the string represents a float or a whole number. The following code will return an error.

x = float("three")

Using str()

The str() constructor casts all data types into a string data type. For example:

x = str("s1") y = str(2) z = str(3.0)

The above code will give the following values:

x = 's1' y = '2' z = '3.0'
Numbers

Python has three number types.

  • int
  • float
  • complex

Python automatically assigns varaible data types.

Integer

An integer is a whole number that is either positive, negative, or zero and can have unlimited length. The following are examples of integers data types.

x = 1 y = 416_583_780 z = 0 2 = -3583

Note that y groups digits using underscores. Grouping digits using underscores makes it easier to read. It is optional and up to the programmer's style.

Float

A float is a number that can be positive or negative and contains one or more decimals. It can also be of any length. The following are examples of float data types.

x = 1.10 y = 1.0 z = -35.59

Complex

Python uses a j to write complex or imaginary numbers instead of the traditional i. The following are examples of complex data types.

x = 3 + 5j y = 5j z = -5j

Cast any number type into complex numbers using complex(). complex(1) will return 1 + 0j. complex(1.4) will return 1.4+ 0j. Complex numbers cannot be recasted into another number type.

Casting between data types can be found in the Changing Data Type section.

Sequences

The sequence data type consists of string, list, and tuple.

  • String- An ordered and immutable sequence of characters.
  • List - An ordered and immutable sequence of values that can have duplicate items.
  • Tuple - An ordered and immutable sequence of values that can have duplicate items.

String

The string data type in Python is a sequence of character data. Single or double quotes can be used around a string. 'hello' and "hello" are the same.

Assigning a string to a variable

Assign a string to a variable by giving a variable name followed by an equal sign and the string in single or double quotes.

a = "hello"

Multi-line strings

Assign a multi-line string to a variable by using three single or double quotes.

a = """A multi-line string can be assigned to the variable using three quotation marks."""

String stored as sequence

Similar to other programming languages, Python stores strings as an array of single characters. Because Python does not have a character data type, it stores a single character as a string with a length of 1. Access any character in a string by calling the index of that character. Note Python calls the first character in a string using index[0] consistent with this data type category.

In a = 'I am a string.', a[3] will return the value m.

List

Lists are a collection of multiple items stored in a single variable. Create lists using square brackets:

myList = [item1, item2, item3]

Items in a list are:

  • Ordered.
  • Mutable.
  • Allowed to have duplicate values

List items are indexed with the first item as index[0].

Lists are ordered

Items in a list have a defined order. Python adds new items to the end of a list.

Lists are mutable

Items in a list can be changed, added, and removed after list creation.

Lists allows duplicate items

Since list items are indexed, a list can have multiple items with the same value.

myList = ["red", "green", "blue", "red", "yellow"]

myList has two "red" values in the list. Each instance of the item "red" has a different index. Call each instance of the item "red" by using the index. myList[0] will call the first "red" item. myList[3] will call the second "red" item.

Data types in lists

List items can be any data type native to Python. It can also contain different data types within the same list. For example:

list1 = [1, 2, 3, 4] list2 = ["she", "sells", "seashells", "by", "the", "seashore"] list3 = [True, False, False] list4 = [1, "she", True, 2, "sells", False, False]

Tuples

Tuples are similar to lists in that it stores multiple ordered items in a single variable. However, a tuple is immutable.

Use round brackets when writing tuples.

myTuple = (item1, item2, item3)

Items in a tuple are:

  • Ordered.
  • Immutable.
  • Allowed to have duplicate values.

Python index tuple items like list items. The first item has index[0] and so on.

Tuples are ordered

Tuples have a defined order that does not change each time you call it.

Tuples are immutable

Items can't be added, removed, or changed after tuple creation.

Tuples allow duplicate items

Like lists, multiple items in a tuple can have the same value because items are indexed.

myTuple = ('a', 'b', 'c', 'b')

myTuple has the value 'b' twice but each instance of 'b'has a different index. Call each instance of 'b' by its index. myTuple[1] will call the first 'b' item and myTuple[3] will call the second 'b' item.

Note, when creating a tuple with one item, add a comma after the item. If the comma is missing, Python will not recognize it as a tuple.

Data types in tuples

Tuple items can have any data type. It can also contain different data types within the same tuple. For example:

tuple1 = [1, 2, 3, 4] tuple2 = ["she", "sells", "seashells", "by", "the", "seashore"] tuple3 = [True, False, False] tuple4 = [1, "she", True, 2, "sells", False, False]
Mapping

In Python, dictionary is the only mapping data type.

Dictionary

Dictionaries help connect pieces of related information. A dictionary is a collection of key:value pairs that pairs a key to a value.

Dictionaries in Python version 3.6 or earlier are unordered but are ordered in version 3.7. We will assume that dictionaries are ordered.

Items in dictionaries are:

  • Ordered.
  • Mutable.
  • Not allowed to have duplicates.

Use curly brackets when writing dictionaries.

myDict = { "red": "apple", "orange": "orange", "purple": "grapes" }

Note that dictionaries can be written horizontally with key:value pairs separated by a comma. However, writing dictionaries vertically makes it easier to read.

Dictionaries are ordered

In Python version 3.7, dictionaries are ordered. In earlier versions of Python, dictionaries are unordered. We will assume that they are ordered here.

Items in a dictionary have a defined order that will not change. Refer to specific items in a dictionary by using its index.

Dictionaries are mutable

Items in a dictionary can be changed, added, or removed after dictionary creation.

Dictionaries don't allow duplicate keys

Each key in the key:value pairs must be unique. Two items in a dictionary can't have the same key. Duplicate values will overwrite the existing values.

myDict = { "brand": "Apple", "model": "iPhone6", "year": 2008, "year": 2010 }

In the above code, the key year will have the value of 2010.

Data types in dictionaries

The data type for the key can be a number, string, float, boolean, tuple, or any built-in object as long as it is immutable. list, dict, or set can't be used as keys in dictionaries.

The data type for values can be any data type.

Sets

Sets, similar to list and tuple, are used to store multiple items in a single variable.

A set is:

  • Unordered.
  • Unindexed.
  • Not allowed to have duplicate values.

Use curly brackets when writing sets.

mySet = {"a", "b", "c"}

Sets are unordered

The items in the set do not have a defined order. The items can appear in a different order each time you call the set. Therefore, the items can't be referred to by their index or key.

Sets are immutable

Items in a set can't be changed after a set's creation. New items can be added.

Sets don't allow duplicates

Sets ignore any duplicate items.

Data types in sets

Set items can be of any data type and can contain different data types.

set1 = {"eggplant", "broccoli", "cauliflower"} set2 = {1, 3, 5, 7} set3 = {True, True, False} set4 = {"a", 36, True, "broccoli", 3}

Note that in set3, Python will ignore the second True when recalling and storing the set.

Boolean

Booleans values are either True or False.

Sometimes, it is helpful to know the Boolean value of a condition or expression. Python can evaluate any expression for its Boolean value. It may be helpful to understand it as a condition.

print(10 > 9) print(10 == 9) print(10 < 9)

The code above will print True, False, and False respectively. Boolean values provide an efficient way to track a particular condition in your program.

Evaluate Boolean values using bool()

The bool() function evaluates any value and gives a return value of True or False.

x = bool("Hello") y = bool(15)

The code above will set both x and y to be True.

When bool() returns True

Almost all values in Python are evaluated to True if it has some content.

  • All strings will return True unless it is an empty string
  • All numbers will return True unless it is 0.
  • All list, tuple, set, and dict will return True, unless it is empty.

When bool() returns False

Few values evaluate to False. Empty values, 0, False, and None will all return False. The examples below will all return False.

bool(Fasle) bool(None) bool(0) bool(" ") bool(()) bool([]) bool({})
